## 📅 Sprint Ceremonies

### Sprint Planning
- **Date:** Tuesday, 20 May 2026
- **Duration:** 2 hours
- **Attendees:** PO, SM, Dev Team
- **Agenda:**
  1. Review Sprint Goal
  2. Review Product Backlog (P0/P1 items)
  3. Select stories for Sprint Backlog
  4. Break stories into tasks
  5. Commit to Sprint Backlog

### Daily Stand-ups
- **Frequency:** Daily (Mon–Fri)
- **Time:** 9:00 AM
- **Format:** What did I do yesterday? What will I do today? Any blockers?
- **Platform:** Slack / Teams / In-person

### Sprint Review
- **Date:** Monday, 2 June 2026
- **Duration:** 1.5 hours
- **Attendees:** PO, SM, Dev Team, Advisory Panel
- **Agenda:**
  1. Demo completed increments
  2. Gather PO feedback
  3. Update Product Backlog based on feedback

### Sprint Retrospective
- **Date:** Monday, 2 June 2026 (after Sprint Review)
- **Duration:** 1 hour
- **Attendees:** SM, Dev Team
- **Agenda:**
  1. What went well?
  2. What could be improved?
  3. What will we commit to improve next sprint?

## ✅ Definition of Done (DoD)

All items must meet these criteria before being marked complete:

- [ ] Code written and reviewed (min. 1 peer review)
- [ ] All acceptance criteria met
- [ ] Component renders correctly in browser
- [ ] No TypeScript errors
- [ ] Responsive (mobile + desktop)
- [ ] Added to staging environment
- [ ] PO has reviewed and accepted
- [ ] Documentation updated
